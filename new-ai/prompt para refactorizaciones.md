1. Analizar la pantalla y su lógica
2. Entender todo lo que usa (hooks, API, contextos, etc.)
3. Crear las entidades del dominio
4. Crear los puertos/use cases
5. Crear la implementación (hooks)
6. Crear los componentes UI (Atomic Design: atoms, molecules, organisms)
7. Ensamblar todo
8. El usuario prueba y confirma si funciona
No crear estructura vacía, no crear re-exports, no separar lógica de UI, hacer una pantalla a la vez.
La instrucción correcta:
"Analizar pantalla → seguir el rastro de la lógica → crear entidades + lógica → crear componentes UI → ensamblar todo → tú pruebas → me confirmas si sigo"
